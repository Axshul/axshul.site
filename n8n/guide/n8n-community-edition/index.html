What Is the Community Edition

When you self-host n8n without purchasing a paid plan, you are running the Community Edition. It is free, source-available under the Sustainable Use License, and includes the vast majority of n8n's feature set.

For most individual developers, freelancers, and small teams building internal automations, the Community Edition is everything you need. You get unlimited workflows, unlimited executions, all 400+ integrations, full webhook support, queue mode, and the complete workflow editor.

Understanding exactly where the Community Edition ends and the paid plans begin is critical before you architect a production system. Discovering a necessary feature is locked after you have built can force a disruptive and expensive migration.

03

What Is Not Included

The Community Edition omissions are deliberate. These features are gated behind paid plans to sustain n8n's development. Here is what each one means in practice:

  • Custom Variables: Instance-wide constants you define once and reference across all workflows. Without this, each workflow must redeclare its own values or pull them from environment variables manually.
  • Environments: Separate staging and production configurations from a single admin panel. Without this, environment separation requires running entirely separate n8n instances.
  • External Secrets: Native integration with HashiCorp Vault, AWS Secrets Manager, or Azure Key Vault. On Community, credentials are stored in n8n's own encrypted database only.
  • External Binary Storage: Route binary files (images, PDFs, attachments) to S3 or compatible object storage. On Community, binaries are stored on the local filesystem, which becomes a scaling bottleneck at high volume.
  • Log Streaming: Push execution logs to Datadog, Splunk, or similar observability platforms in real time. Basic file-based logging is included, but forwarding is not.
  • Multi-Main Mode: Run multiple n8n main processes simultaneously for high-availability failover. Queue mode (single main, multiple workers) is included. Active-active redundancy is Enterprise-only.
  • Projects and Sharing: Workspace-level organization with fine-grained access control. On Community, only the instance owner and the workflow creator can access a given workflow.
  • SSO (SAML / LDAP): Single Sign-On with enterprise identity providers. Community users must authenticate with local n8n accounts only.
  • Git Version Control: Built-in Git integration for pushing workflows to a repository and switching branches. Manual JSON exports remain available as a workaround.
04

The Registered Edition

n8n offers a genuinely compelling middle ground: by registering your self-hosted instance with your email address, you unlock three additional features completely free of charge. The license does not expire once activated.

Three features unlocked by registration:

  • Folders - Organize workflows into a hierarchical folder structure. Essential once an instance grows beyond 20 to 30 workflows.
  • Debug in Editor - Copy and pin execution data from a previous run into an active workflow session. Replay problematic payloads without re-triggering the workflow.
  • Custom Execution Data - Attach, search, and annotate custom metadata to any execution. Build audit trails, tag executions with customer IDs, or flag runs for review.

Registration requires only a valid email address. n8n uses it to send you the license key and occasional product updates. No credit card, no phone number, no company information required.

Once activated, the license is permanent. n8n may adjust the set of features included in the registered tier in the future, but previously unlocked features will not be revoked from existing activations.

05

How to Register

New instance: The registration option is presented during the initial account creation flow. Enter your email when prompted and a license key is sent immediately.

Existing instance: If your instance is already running without registration, activating the free features takes under two minutes:

  1. Click the three-dot menu in the lower-left corner of the n8n editor.
  2. Select Settings, then navigate to Usage and plan.
  3. Click Unlock, enter your email address, and select Send me a free license key.
  4. Check your inbox for the license email from n8n.
  5. Click the activation button in the email, or return to Settings > Usage and plan and select Enter activation key.

Heads up: If you are running n8n behind a strict firewall that blocks outbound connections, the activation request may fail. Ensure your server can reach license.n8n.io on port 443.

When to Upgrade

The Community Edition handles the overwhelming majority of real-world automation use cases. The decision to upgrade typically comes down to one of three specific requirements:

  • Multi-user team access with permissions. If more than one person needs to own, edit, or share specific workflows independently, you need at least a paid plan for the Projects and Sharing features. On Community, only the owner and creator have access.
  • Enterprise identity and compliance. If your organization mandates SSO, LDAP, external secret management, or audit log streaming to a SIEM, the Enterprise plan addresses all of these. These are hard requirements you cannot work around.
  • Environment promotion pipelines. If your team operates a formal staging-to-production workflow promotion cycle and requires Git-based version control of workflow definitions, the Enterprise Git integration is the only native solution.

For everything else, a registered Community Edition instance is a fully capable foundation for serious production automation. Many teams run hundreds of workflows on Community without hitting any functional limits.

Cost context: n8n Enterprise is custom-priced and typically starts at several hundred dollars per month. For pricing details and plan breakdowns, see our n8n Cloud Plans guide. For licensing details, see n8n License and Regulations.

Production Usage Tips

Running Community Edition in production is entirely viable. Here are the practical recommendations that matter most:

  • Use PostgreSQL, not SQLite. The default SQLite backend works for testing but will corrupt or lock under concurrent writes. Switch to PostgreSQL with DB_TYPE=postgresdb before you go live. See our VPS self-hosting guide for the full Docker Compose setup.
  • Set the encryption key once, back it up. N8N_ENCRYPTION_KEY encrypts all saved credentials. Generate it with openssl rand -hex 16, store it somewhere safe, and never change it. If you lose this key, all credentials become unrecoverable.
  • Prune execution data. n8n stores every execution by default. On a busy instance, the database grows quickly. Set EXECUTIONS_DATA_MAX_AGE (in hours) to auto-prune old executions. 720 hours (30 days) is a reasonable default.
  • Enable queue mode for heavy workloads. If you process thousands of executions daily, enable queue mode with Redis to separate the main process from worker processes. This prevents a long-running workflow from blocking the editor or webhook triggers.
  • Automate backups. A daily PostgreSQL dump and n8n data volume backup is the minimum. Use cron and store backups offsite. Losing a production n8n instance with months of workflow development is painful and entirely preventable.
  • Pin the Docker image version. Using n8nio/n8n:latest in production means any restart could pull a new version with breaking changes. Pin to a specific version like n8nio/n8n:1.52.0 and upgrade deliberately.

FAQ

Can I use the Community Edition for commercial projects?

Yes. The Sustainable Use License allows you to use n8n for any internal business automation. The restriction is against reselling n8n itself as a product or building a competing workflow automation platform. Using n8n to automate your own business operations, serve clients, or process customer data is fully permitted.

Does registration send my workflow data to n8n?

No. Registration sends your email address to n8n's license server and receives a license key in return. Your workflow definitions, credentials, and execution data never leave your server. n8n collects anonymized telemetry (node types used, execution counts) by default, which you can disable with N8N_DIAGNOSTICS_ENABLED=false.

What happens if the registration license server goes down?

Once activated, the license is cached locally. Your registered features continue working even if n8n's license server is unreachable. The license only needs to phone home periodically for validation. Extended outages would not immediately affect your instance.

Can I have multiple users on Community Edition?

Yes, you can invite multiple users to your Community instance. However, there is no role-based access control or workflow sharing. Each user can only see and edit workflows they created, plus the instance owner can see everything. For fine-grained permissions and project-based access, you need Enterprise.

Is there a workflow or node limit on Community?

No. There are no artificial limits on the number of workflows, nodes per workflow, or executions. The only limits are your server's hardware resources. This is one of the strongest arguments for self-hosting over cloud plans, which impose execution quotas.

Can I workaround the missing Git integration?

Yes. The n8n CLI supports exporting workflows as JSON files: n8n export:workflow --all --output=./backups/. You can commit these JSON files to a Git repository manually or via a cron job. This is not as elegant as the native Git integration (no branch switching, no diff view in the UI), but it provides version history and disaster recovery.
